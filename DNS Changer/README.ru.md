# DNS Changer

Утилита командной строки на C++, позволяющая изменять и восстанавливать параметры DNS на системах Linux.

## Возможности

- **Изменение DNS**: Установка пользовательского адреса DNS сервера
- **Восстановление DNS**: Восстановление предыдущих параметров DNS из резервной копии
- **Валидация**: Проверка формата IP адреса для обеспечения корректности адреса DNS сервера
- **Резервное копирование**: Автоматическое сохранение текущих параметров DNS перед изменениями
- **Проверка привилегий**: Проверка наличия прав администратора (root)

## Требования

- Операционная система Linux
- Компилятор C++17 или новее
- Привилегии root/sudo для модификации параметров DNS
- CMake 3.20 или новее (для сборки)

## Сборка

Использование CMake:

```bash
cd "DNS Changer"
mkdir build
cd build
cmake ..
make
```

Или прямая компиляция:

```bash
g++ -std=c++17 src/main.cpp -o my_program
```

## Использование

Программа должна запускаться с правами администратора:

```bash
sudo ./my_program set <dns_ip>
```

```bash
sudo ./my_program restore
```

### Команды

- **`set <dns_ip>`**: Изменяет DNS сервер на указанный IP адрес
  - Пример: `sudo ./my_program set 8.8.8.8`
  - Автоматически создает резервную копию текущих параметров DNS в `/etc/resolv.conf.backup`

- **`restore`**: Восстанавливает параметры DNS из файла резервной копии
  - Пример: `sudo ./my_program restore`
  - Восстанавливает из `/etc/resolv.conf.backup`

## Примеры

Изменение DNS на публичный DNS Google:
```bash
sudo ./my_program set 8.8.8.8
```

Изменение DNS на DNS Cloudflare:
```bash
sudo ./my_program set 1.1.1.1
```

Восстановление предыдущих параметров DNS:
```bash
sudo ./my_program restore
```

## Как это работает

### Установка DNS
1. Проверяет наличие прав администратора (root)
2. Валидирует формат IP адреса (проверяет корректность IPv4)
3. Создает резервную копию текущего файла `/etc/resolv.conf` в `/etc/resolv.conf.backup`
4. Заменяет параметры DNS в файле `/etc/resolv.conf` на новый сервер

### Восстановление DNS
1. Проверяет наличие прав администратора (root)
2. Читает файл резервной копии из `/etc/resolv.conf.backup`
3. Восстанавливает исходные параметры DNS в `/etc/resolv.conf`

## Валидация IP адресов

Программа проверяет IPv4 адреса по следующим критериям:
- Должны быть 4 октета, разделенные точками
- Каждый октет должен быть числом от 0 до 255

Примеры валидных адресов: `8.8.8.8`, `1.1.1.1`, `192.168.1.1`
Примеры невалидных адресов: `256.1.1.1`, `8.8.8`, `invalid.address`

## Структура проекта

### Директории

#### `/src` - Исходный код
- `main.cpp` - Исходный код основной программы
  - Функции валидации IP адресов
  - Функции резервного копирования и восстановления DNS
  - Проверка прав администратора
  - Операции ввода-вывода для конфигурации DNS

#### `/build` - Скомпилированные бинарные файлы
- `my_program.bin` - Исполняемый файл (Linux)

### Системные файлы

- `/etc/resolv.conf` - Файл конфигурации DNS системы (модифицируется программой)
- `/etc/resolv.conf.backup` - Резервная копия параметров DNS, созданная при выполнении команды `set`

## Безопасность

- Резервные копии создаются автоматически перед модификацией параметров DNS
- Валидация IP адреса предотвращает внесение невалидных записей
- Требует прав администратора для предотвращения несанкционированных изменений
- Исходные параметры DNS всегда можно восстановить

## Решение проблем

**Ошибка: "Run as root!"**
- Решение: Используйте `sudo` при запуске программы

**Ошибка: "Cannot open /etc/resolv.conf for reading"**
- Решение: Убедитесь, что файл существует и программа имеет права на чтение

**Ошибка: "Invalid DNS IP address"**
- Решение: Проверьте, что IP адрес имеет корректный формат IPv4 (например, `8.8.8.8`)

**Ошибка: "Backup file not found"**
- Решение: Резервная копия никогда не была создана. Сначала выполните команду `set` для создания резервной копии.

## Примечания

- Изменения DNS могут занять некоторое время для распространения
- Программа напрямую модифицирует файл `/etc/resolv.conf`, что требует прав администратора
- На некоторых системах параметры DNS могут управляться NetworkManager или systemd-resolved. Этот инструмент напрямую модифицирует `/etc/resolv.conf`, что может быть переопределено этими сервисами
