# DNS Changer

Мощная утилита для управления DNS на системах Linux с интерфейсом командной строки (CLI) и графическим интерфейсом (GUI).

## Возможности

- **Изменение DNS**: Установка пользовательского адреса DNS сервера
- **Восстановление DNS**: Восстановление предыдущих параметров DNS из резервной копии
- **Проверка статуса**: Просмотр текущей конфигурации DNS
- **Валидация**: Проверка формата IP адреса для обеспечения корректности адреса DNS сервера
- **Резервное копирование**: Автоматическое сохранение текущих параметров DNS перед изменениями
- **Поддержка нескольких менеджеров**: Работает с NetworkManager, systemd-resolved, resolvconf и openresolv
- **GUI приложение**: Удобный графический интерфейс на базе CustomTkinter
- **Быстрые пресеты DNS**: Доступ в один клик к популярным DNS провайдерам (Cloudflare, Google, Quad9)

## Требования

### Для CLI (интерфейс командной строки)
- Операционная система Linux
- Компилятор C++17 или новее (g++)
- Привилегии root/sudo для модификации параметров DNS

### Для GUI (графический интерфейс)
- Python 3.7 или новее
- Библиотека CustomTkinter (см. установку ниже)

## Установка

### Быстрая установка (рекомендуется)

Запустите скрипт установки от имени root:

```bash
sudo ./install.sh
```

Это скомпилирует программу и установит её в `/usr/local/bin/dnschanger`.

### Ручная компиляция

Компиляция напрямую с помощью g++:

```bash
g++ -std=c++17 -O2 src/main.cpp -o dnschanger
sudo cp dnschanger /usr/local/bin/
```

### Настройка GUI

Установите зависимости Python:

```bash
pip3 install -r gui/requirements.txt
```

Или установите вручную:

```bash
pip3 install customtkinter
```

## Использование

### Интерфейс командной строки (CLI)

Программа должна запускаться с правами администратора:

```bash
sudo dnschanger set <dns_ip>    # Установить DNS сервер
sudo dnschanger restore         # Восстановить предыдущий DNS
sudo dnschanger status          # Проверить текущий DNS
```

### Графический интерфейс (GUI)

Запустите GUI приложение:

```bash
sudo python3 gui/gui.py
```

GUI предоставляет:
- **Кнопки быстрого DNS** для популярных провайдеров (Cloudflare, Google, Quad9)
- **Ввод пользовательского DNS** для любого DNS сервера
- **Функцию восстановления** для возврата к предыдущим настройкам
- **Проверку статуса** для просмотра текущей конфигурации DNS
- **Вывод в реальном времени** с отображением выполнения команд и результатов

### Команды

- **`set <dns_ip>`**: Изменяет DNS сервер на указанный IP адрес
  - Пример: `sudo dnschanger set 8.8.8.8`
  - Автоматически создает резервную копию текущих параметров DNS в `/etc/resolv.conf.backup`

- **`restore`**: Восстанавливает параметры DNS из файла резервной копии
  - Пример: `sudo dnschanger restore`
  - Восстанавливает из `/etc/resolv.conf.backup`

- **`status`**: Отображает текущую конфигурацию DNS
  - Пример: `sudo dnschanger status`

2. Нажмите одну из кнопок быстрого DNS (Cloudflare, Google, Quad9) или введите пользовательский DNS IP
3. Нажмите "Apply DNS" для установки пользовательского DNS
4. Используйте "Check current DNS" для проверки конфигурации
5. Нажмите "Restore DNS" для возврата к предыдущим настройкам

## Как это работает

### Определение сетевого менеджера

Программа автоматически определяет, какая система управления сетью работает на вашем дистрибутиве Linux:

1. **NetworkManager** (nmcli) - Распространён на Ubuntu, Fedora и многих современных дистрибутивах
2. **systemd-resolved** - Используется в дистрибутивах на базе systemd
3. **resolvconf** - Традиционные системы Debian/Ubuntu
4. **openresolv** - Альтернативное управление резолвером
5. **Fallback** - Прямая модификация `/etc/resolv.conf`, если менеджер не обнаружен

### Установка DNS
1. Проверяет наличие прав администратора (root)
2. Определяет активный сетевой менеджер
3. Валидирует формат IP адреса (проверяет корректность IPv4)
4. Создает резервную копию текущей конфигурации DNS в `/etc/resolv.conf.backup`
5. Применяет новые настройки DNS используя соответствующий метод для обнаруженного сетевого менеджера
6. При неудаче автоматически восстанавливает предыдущую конфигурацию

### Восстановление DNS
1. Проверяет наличие прав администратора (root)
2. Определяет активный сетевой менеджер
3. Читает файл резервной копии из `/etc/resolv.conf.backup`
4. Восстанавливает исходные параметры DNS используя соответствующий метод

### Проверка статуса
1. Проверяет наличие прав администратора (root)
2. Отображает текущую конфигурацию DNS на основе обнаруженного сетевого менеджера

## Валидация IP адресов

Программа проверяет IPv4 адреса по следующим критериям:
- Должны быть 4 октета, разделенные точками
- Каждый октет должен быть числом от 0 до 255

Примеры валидных адресов: `8.8.8.8`, `1.1.1.1`, `192.168.1.1`
Примеры невалидных адресов: `256.1.1.1`, `8.8.8`, `invalid.address`

## Решение проблем

**Ошибка: "This program must be run as root!"**
- Решение: Используйте `sudo` при запуске программы
- Пример: `sudo dnschanger set 8.8.8.8`

**Ошибка: "The 'dnschanger' command was not found in PATH" (GUI)**
- Решение: Убедитесь, что программа установлена с помощью `sudo ./install.sh`
- Или убедитесь, что скомпилированный бинарный файл находится в вашем PATH

**Ошибка: "Cannot open /etc/resolv.conf for reading"**
- Решение: Убедитесь, что файл существует и программа имеет права на чтение

**Ошибка: "Invalid DNS IP address"**
- Решение: Проверьте, что IP адрес имеет корректный формат IPv4 (например, `8.8.8.8`)

**Ошибка: "Backup file not found"**
- Решение: Резервная копия никогда не была создана. Сначала выполните команду `set` для создания резервной копии.

**GUI не запускается**
- Решение: Установите зависимости Python с помощью `pip3 install -r gui/requirements.txt`
- Убедитесь, что запускаете с sudo: `sudo python3 gui/gui.py`

**Изменения DNS не сохраняются после перезагрузки**
- Некоторые сетевые менеджеры могут переопределять ручные настройки DNS
- Рассмотрите использование GUI вашего сетевого менеджера или конфигурационных файлов для постоянных изменений
