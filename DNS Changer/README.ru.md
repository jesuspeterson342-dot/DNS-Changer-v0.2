# DNS Changer

Мощная и интеллектуальная утилита для управления DNS на системах Linux с интерфейсом командной строки (CLI) и графическим интерфейсом (GUI). Автоматически определяет сетевой менеджер вашей системы и применяет изменения DNS используя соответствующий метод.

## Возможности

- **🔄 Изменение DNS**: Установка пользовательского адреса DNS сервера с автоматической валидацией
- **♻️ Восстановление DNS**: Восстановление предыдущих параметров DNS из автоматических резервных копий
- **📊 Проверка статуса**: Просмотр текущей конфигурации DNS в понятном формате
- **✅ Валидация IP**: Надёжная проверка IPv4 адресов для обеспечения корректности адреса DNS сервера
- **💾 Автоматическое резервное копирование**: Создание резервных копий перед любыми изменениями для предотвращения потери конфигурации
- **🔧 Поддержка нескольких менеджеров**: Интеллектуальное определение и поддержка:
  - NetworkManager (nmcli) - Современные дистрибутивы Linux
  - systemd-resolved - Системы на базе systemd
  - resolvconf - Традиционные системы Debian/Ubuntu
  - openresolv - Альтернативное управление резолвером
  - Режим Fallback - Прямая модификация `/etc/resolv.conf`
- **🎨 GUI приложение**: Современный, удобный графический интерфейс на базе CustomTkinter
- **⚡ Быстрые пресеты DNS**: Доступ в один клик к популярным DNS провайдерам (Cloudflare, Google, Quad9)
- **🛡️ Безопасность прежде всего**: Автоматический откат при сбое для предотвращения поломки DNS
- **🖥️ Интеграция с рабочим столом**: Запуск из меню приложений с помощью desktop entry файла

## Требования

### Для CLI (интерфейс командной строки)
- Операционная система Linux
- Компилятор C++17 или новее (g++)
- Привилегии root/sudo для модификации параметров DNS

### Для GUI (графический интерфейс)
- Python 3.7 или новее
- Библиотека CustomTkinter (см. установку ниже)

## Установка

### Быстрая установка (рекомендуется)

Запустите скрипт установки от имени root:

```bash
sudo ./install.sh
```

Это выполнит:
- Компиляцию C++ программы с оптимизацией
- Установку CLI бинарного файла в `/usr/local/bin/dnschanger`
- Установку GUI в `/usr/local/share/dnschanger/`
- Создание desktop entry для интеграции с меню приложений

### Использование Makefile

Альтернативно, используйте предоставленный Makefile:

```bash
make              # Компиляция программы
sudo make install # Установка CLI, GUI и desktop entry
sudo make uninstall # Удаление всех установленных файлов
```

### Ручная компиляция

Компиляция напрямую с помощью g++:

```bash
g++ -std=c++17 -O2 -Wall src/main.cpp -o dnschanger
sudo cp dnschanger /usr/local/bin/
```

### Настройка GUI

Установите зависимости Python (требуется для GUI):

```bash
pip3 install -r gui/requirements.txt
```

Или установите вручную:

```bash
pip3 install customtkinter
```

**Рекомендуется**: Использовать виртуальное окружение:

```bash
python3 -m venv venv
source venv/bin/activate
pip install customtkinter
```

## Использование

### Интерфейс командной строки (CLI)

Программа должна запускаться с правами администратора:

```bash
sudo dnschanger set <dns_ip>    # Установить DNS сервер
sudo dnschanger restore         # Восстановить предыдущий DNS
sudo dnschanger status          # Проверить текущий DNS
```

### Графический интерфейс (GUI)

**Вариант 1**: Используйте удобный скрипт запуска (рекомендуется):

```bash
./run_gui.sh
```

Этот скрипт автоматически:
- Проверяет наличие Python и зависимостей
- Предоставляет доступ к X11 для root
- Запускает GUI с правильным окружением
- Очищает разрешения X11 после выхода

**Вариант 2**: Запуск напрямую:

```bash

```

**Вариант 3**: Запуск из меню приложений (после установки):

Найдите "DNS Changer" в меню приложений.

GUI предоставляет:
- **Кнопки быстрого DNS** для популярных провайдеров (Cloudflare 1.1.1.1, Google 8.8.8.8, Quad9 9.9.9.9)
- **Ввод пользовательского DNS** с валидацией IP в реальном времени
- **Функцию восстановления** для возврата к предыдущим настройкам
- **Проверку статуса** для просмотра текущей конфигурации DNS
- **Вывод в реальном времени** с отображением выполнения команд и результатов
- **Современную тёмную тему** на базе CustomTkinter
- **Автоматическое определение исполняемого файла** (находит dnschanger в нескольких местах)

### Команды

- **`set <dns_ip>`**: Изменяет DNS сервер на указанный IP адрес
  - Пример: `sudo dnschanger set 8.8.8.8`
  - Автоматически создает резервную копию текущих параметров DNS в `/etc/resolv.conf.backup`

- **`restore`**: Восстанавливает параметры DNS из файла резервной копии
  - Пример: `sudo dnschanger restore`
  - Восстанавливает из `/etc/resolv.conf.backup`

- **`status`**: Отображает текущую конфигурацию DNS
  - Пример: `sudo dnschanger status`

## Как это работает

### Определение сетевого менеджера

Программа использует интеллектуальное определение для идентификации системы управления сетью на вашем дистрибутиве Linux:

1. **NetworkManager** (nmcli) - Определяется проверкой:
   - `systemctl is-active NetworkManager`
   - Наличие `/run/NetworkManager/NetworkManager.pid`

2. **systemd-resolved** - Определяется проверкой:
   - `systemctl is-active systemd-resolved`
   - Наличие `/run/systemd/resolve/stub-resolv.conf` или `/run/systemd/resolve/resolv.conf`

3. **openresolv** - Определяется проверкой:
   - Наличие `/etc/resolvconf.conf`
   - Конфигурация содержит директиву `name_servers`

4. **resolvconf** (Debian/Ubuntu) - Определяется проверкой:
   - Наличие `/run/resolvconf/resolv.conf`

5. **Fallback** - Прямая модификация `/etc/resolv.conf`, если менеджер не обнаружен

### Установка DNS (команда set)
1. ✅ Проверяет наличие прав администратора (root)
2. 🔍 Определяет активный сетевой менеджер
3. ✅ Валидирует формат IP адреса (IPv4: 4 октета, каждый 0-255)
4. 💾 Создает резервную копию текущей конфигурации DNS в `/etc/resolv.conf.backup`
5. 🔧 Применяет новые настройки DNS используя соответствующий метод:
   - **NetworkManager**: Использует `nmcli dev modify` и `nmcli con reload`
   - **systemd-resolved**: Модифицирует `/etc/systemd/resolved.conf` и перезапускает сервис
   - **resolvconf**: Записывает в `/etc/resolvconf/resolv.conf.d/head` и выполняет `resolvconf -u`
   - **openresolv**: Обновляет `/etc/resolvconf.conf` и выполняет `resolvconf -u`
   - **Fallback**: Напрямую записывает в `/etc/resolv.conf`
6. 🛡️ При неудаче автоматически восстанавливает предыдущую конфигурацию

### Восстановление DNS (команда restore)
1. ✅ Проверяет наличие прав администратора (root)
2. 🔍 Определяет активный сетевой менеджер
3. 📂 Читает файл резервной копии из `/etc/resolv.conf.backup`
4. ✅ Проверяет существование и содержимое файла резервной копии
5. ♻️ Восстанавливает исходные параметры DNS используя соответствующий метод для обнаруженного менеджера

### Проверка статуса (команда status)
1. ✅ Проверяет наличие прав администратора (root)
2. 🔍 Определяет активный сетевой менеджер
3. 📊 Отображает текущую конфигурацию DNS:
   - **NetworkManager**: Показывает DNS из `nmcli -t -f IP4.DNS dev show`
   - **systemd-resolved**: Показывает вывод `resolvectl status` или читает `/etc/systemd/resolved.conf`
   - **openresolv**: Показывает `name_servers` из `/etc/resolvconf.conf` и текущий `/etc/resolv.conf`
   - **resolvconf/Fallback**: Парсит и отображает nameservers из `/etc/resolv.conf`

## Валидация IP адресов

Программа проверяет IPv4 адреса по следующим критериям:
- Должны быть 4 октета, разделенные точками
- Каждый октет должен быть числом от 0 до 255

Примеры валидных адресов: `8.8.8.8`, `1.1.1.1`, `192.168.1.1`
Примеры невалидных адресов: `256.1.1.1`, `8.8.8`, `invalid.address`

## Решение проблем

### Проблемы CLI

**Ошибка: "This program must be run as root!"**
- **Причина**: Программа требует права администратора для модификации системных настроек DNS
- **Решение**: Используйте `sudo` при запуске программы
- **Пример**: `sudo dnschanger set 8.8.8.8`

**Ошибка: "Invalid DNS IP address"**
- **Причина**: Предоставленный IP адрес не соответствует формату IPv4
- **Решение**: Проверьте, что IP адрес корректен (например, `8.8.8.8`)
- **Корректный формат**: 4 октета, разделённые точками, каждый октет 0-255
- **Некорректные примеры**: `256.1.1.1`, `8.8.8`, `invalid.address`

**Ошибка: "Backup file not found"**
- **Причина**: Резервная копия не существует в `/etc/resolv.conf.backup`
- **Решение**: Сначала выполните команду `set` для создания резервной копии перед восстановлением

**Ошибка: "Cannot open /etc/resolv.conf for reading"**
- **Причина**: Файл не существует или отсутствуют права на чтение
- **Решение**: Убедитесь, что файл существует и программа имеет права на чтение

**Ошибка: "Failed to set DNS via nmcli"**
- **Причина**: Команда NetworkManager завершилась с ошибкой
- **Решение**: Проверьте, что NetworkManager запущен: `systemctl status NetworkManager`

### Проблемы GUI

**Ошибка: "dnschanger executable not found!"**
- **Причина**: CLI программа не установлена или не находится в PATH
- **Решение**:
  - Установите с помощью `sudo ./install.sh`
  - Или скомпилируйте вручную: `g++ -std=c++17 -O2 src/main.cpp -o dnschanger`
  - Убедитесь, что файл находится в `/usr/local/bin/` или другом месте в PATH

**Ошибка: "customtkinter module not found!"**
- **Причина**: Зависимость Python не установлена
- **Решение**: Установите с помощью `pip3 install customtkinter` или `pip3 install -r gui/requirements.txt`

**Ошибка: "Root privileges required!"**
- **Причина**: GUI не запущен с sudo
- **Решение**: Запустите с `sudo python3 gui/gui.py` или используйте `./run_gui.sh`

**Ошибка: "No DISPLAY environment variable!"**
- **Причина**: X11 дисплей недоступен для root
- **Решение**:
  - Используйте `sudo -E python3 gui/gui.py` для сохранения окружения
  - Или предоставьте доступ к X11: `xhost +si:localuser:root`
  - Или используйте скрипт запуска: `./run_gui.sh`

**Окно GUI не появляется**
- **Причина**: Проблемы с разрешениями X11
- **Решение**:
  ```bash
  xhost +si:localuser:root
  sudo -E python3 gui/gui.py
  ```

### Общие проблемы

**Изменения DNS не сохраняются после перезагрузки**
- **Причина**: Сетевой менеджер может переопределять ручные настройки
- **Решение**:
  - Для постоянных изменений настройте сетевой менеджер напрямую
  - NetworkManager: Используйте `nmcli` или GUI настроек сети
  - systemd-resolved: Отредактируйте `/etc/systemd/resolved.conf` постоянно

**Изменения не вступают в силу немедленно**
- **Причина**: Кэш DNS или кэширование на уровне приложения
- **Решение**:
  - Очистите кэш DNS: `sudo systemd-resolve --flush-caches` (если используется systemd-resolved)
  - Перезапустите сеть: `sudo systemctl restart NetworkManager`
  - Проверьте с помощью: `nslookup google.com` или `dig google.com`

**"Warning: Unknown network manager"**
- **Причина**: Не обнаружен распознаваемый сетевой менеджер
- **Эффект**: Программа будет использовать fallback метод (прямая модификация `/etc/resolv.conf`)
- **Примечание**: Это не ошибка, просто информационное сообщение

## Примечания

- Изменения DNS могут занять некоторое время для распространения
- Программа автоматически определяет и работает с сетевым менеджером вашей системы
- Поддерживаемые сетевые менеджеры: NetworkManager, systemd-resolved, resolvconf, openresolv
- Всегда создаёт резервную копию перед внесением изменений для безопасности
- GUI требует установки CLI инструмента и его доступности в PATH

## Соображения безопасности

- Всегда проверяйте адреса DNS серверов перед их применением
- Используйте доверенные DNS провайдеры, чтобы избежать DNS hijacking
- Программа требует прав администратора для модификации системных настроек DNS
- Резервные копии хранятся в `/etc/resolv.conf.backup` - храните этот файл в безопасности
